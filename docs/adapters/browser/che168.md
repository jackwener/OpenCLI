# 车168 / 汽车之家二手车 che168

**Mode**: 🌐 Public (`spec`) · 🔐 Browser (`browse` / `car`) · **Domain**: `che168.com`

che168 (二手车之家, the used-car arm of the 汽车之家 / Autohome group) has two
very different surfaces:

- **Spec/config data** is served by an open, login-free cache API
  (`cacheapigo.che168.com/CarProduct/GetParam.ashx`, GBK JSON). `spec` reads it
  directly — no browser.
- **Used-car listing & detail pages** (`www.che168.com`) are gated behind an
  anti-bot JS challenge; a plain HTTP client only ever gets the challenge shell.
  So `browse` / `car` run in the logged-in browser, where the challenge clears,
  and extract the rendered DOM.

## Commands

| Command | Mode | Description |
|---------|------|-------------|
| `opencli che168 spec <specid>` | 🌐 Public | Full car trim parameter sheet → 基本参数 / 车身 / 发动机 / 变速箱 / 底盘 / 制动 |
| `opencli che168 browse [city]` | 🔐 Browser | Used cars for sale in a city → price / registration / mileage |
| `opencli che168 car <url>` | 🔐 Browser | One listing's detail → price, down payment, registration, mileage, gearbox, transfers, location |

- `spec` takes a **specid** (the autohome/che168 trim id) — a bare number or any
  `.../spec/<id>/` URL / `?specid=<id>` query.
- `car` takes the listing **url** from `browse` (its `url` column). che168's
  desktop detail page needs the dealer-qualified path
  (`/dealer/<dealer>/<infoid>.html`); a bare info id can't reach it, so the full
  URL is required.

## Usage Examples

```bash
# Full spec/config sheet for a trim (open API, no browser)
opencli che168 spec 39616
opencli che168 spec https://www.autohome.com.cn/spec/39616/
opencli che168 spec 39616 -f json

# Browse used-car listings (defaults to Beijing)
opencli che168 browse
opencli che168 browse 上海 --limit 30
opencli che168 browse shanghai           # city pinyin also works

# One listing in detail — pass the url from browse
opencli che168 car https://www.che168.com/dealer/649418/58405825.html
```

## Output Columns

| Command | Columns |
|---------|---------|
| `spec` | `group, field, value` |
| `browse` | `rank, info_id, title, price, reg_date, mileage, city, url` |
| `car` | `field, value` (title, price, down_payment, reg_date, mileage, gearbox, fuel, transfers, location, publish_date, url) |

## Cities (`browse`)

Pass a Chinese city name, a 2-letter code, or the che168 pinyin slug. Known:
北京(beijing), 上海(shanghai), 广州(guangzhou), 深圳(shenzhen), 杭州(hangzhou),
成都(chengdu), 重庆(chongqing), 南京(nanjing), 武汉(wuhan), 天津(tianjin),
西安(xian), 苏州(suzhou), 长沙(changsha), 青岛(qingdao), 郑州(zhengzhou),
合肥(hefei). Any other pinyin slug is passed through as-is.

## Notes & Limits

- **`browse` returns the first list page** (~50 server-rendered listings) for a
  city. Deep pagination and keyword/brand filters live behind che168's signed
  internal endpoints and are intentionally not faked.
- **`car` needs the dealer-qualified URL** from `browse`. A bare info id
  redirects che168 to an error page, so the command fails loudly with an
  argument error rather than returning blanks.
- **Fields are surfaced honestly.** A field absent on a given listing (e.g.
  `fuel` on some pages) comes back empty rather than fabricated.
- If che168 ever puts the listing pages fully behind the challenge even in the
  browser, `browse` / `car` fail with an auth-required error.

## Prerequisites

- `spec`: none — open API.
- `browse` / `car`: a running browser bridge (`opencli doctor`). No che168 login
  is required; the browser is used only to clear the anti-bot challenge.
